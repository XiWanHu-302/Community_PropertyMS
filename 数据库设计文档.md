```
┌──────────────────────────────────────────────────┐
```

### 关系说明

| 关系                                     | 说明            |
| -------------------------------------- | ------------- |
| `building` 1:N `household`             | 一栋楼有多个住户      |
| `household` 1:N `property_fee`         | 一户有多条物业费月记录   |
| `household` 1:N `parking_space`        | 一户可租多个停车位     |
| `parking_space` 1:N `parking_fee`      | 一个车位有多条停车费月记录 |
| `household` 1:N `repair`               | 一户可多次报修       |
| `household` 1:1 `user` (role=resident) | 一户一个业主账号      |
| `user` (role=admin/maintenance) 不关联住户  | 管理员和维修员独立账号   |

## 四、表结构详细设计

### 4.1 `building` — 楼栋信息表

| 字段名               | 类型          | 约束                         | 说明                 |
| ----------------- | ----------- | -------------------------- | ------------------ |
| `building_no`     | VARCHAR(10) | **PK**                     | 楼号（如 "28"），唯一标识一栋楼 |
| `floor_count`     | INT         | **NOT NULL**, CHECK ≥ 1    | 总层数                |
| `units_per_floor` | INT         | **NOT NULL**, CHECK ≥ 1    | 每层户数               |
| `create_time`     | DATETIME    | DEFAULT CURRENT\_TIMESTAMP | 创建时间               |

**约束说明：**

- `building_no` 作为主键，直接使用楼号，不另设自增ID
- `floor_count` 和 `units_per_floor` 必须 ≥ 1

**用途：** 房号合法性校验。如楼栋 28 号有 28 层、每层 4 户，则层号范围 1\~28，户号范围 1\~4。房号 "1302" 对应 13 层 02 户号，需满足 13 ≤ 28 且 2 ≤ 4。

***

### 4.2 `household` — 住户信息表

| 字段名                   | 类型            | 约束                                            | 说明           |
| --------------------- | ------------- | --------------------------------------------- | ------------ |
| `household_id`        | INT           | **PK**, AUTO\_INCREMENT                       | 住户ID         |
| `building_no`         | VARCHAR(10)   | **FK** → building(building\_no), **NOT NULL** | 所属楼栋（楼号）     |
| `floor_no`            | INT           | **NOT NULL**                                  | 所在层号         |
| `unit_no`             | INT           | **NOT NULL**                                  | 户号（该层第几户）    |
| `area`                | DECIMAL(10,2) | **NOT NULL**, CHECK > 0                       | 房屋面积（平方米）    |
| `property_fee_rate`   | DECIMAL(10,2) | **NOT NULL**, CHECK > 0                       | 物业费单价（元/㎡/月） |
| `owner_name`          | VARCHAR(50)   | **NOT NULL**                                  | 户主姓名         |
| `phone`               | VARCHAR(20)   | **NOT NULL**                                  | 联系电话         |
| `work_unit`           | VARCHAR(100)  | <br />                                        | 工作单位         |
| `family_size`         | INT           | DEFAULT 1                                     | 家庭人数         |
| `repair_fund_balance` | DECIMAL(12,2) | DEFAULT 0                                     | 维修基金余额（元）    |
| `status`              | TINYINT       | DEFAULT 1, CHECK IN (0,1)                     | 1=在住, 0=已搬离  |
| `check_in_date`       | DATE          | <br />                                        | 入住日期         |
| `check_out_date`      | DATE          | <br />                                        | 搬离日期         |
| `create_time`         | DATETIME      | DEFAULT CURRENT\_TIMESTAMP                    | 创建时间         |

**约束说明：**

- **CHECK约束：** `area > 0`, `property_fee_rate > 0`, `status IN (0,1)`
- **外键约束：** `building_no` 引用 `building(building_no)`，`ON DELETE RESTRICT` 禁止删除有住户的楼栋
- **房号显示：** 前端拼接显示为 `CONCAT(floor_no, LPAD(unit_no, 2, '0'))`，如 13楼2户号 → "1302"
- **房号合法性校验（应用层）：** `floor_no ≤ building.floor_count AND unit_no ≤ building.units_per_floor`
- **房间唯一性（应用层）：** 同一房间只允许一户**在住**（`status=1`），已搬离的不限制。因此同一房间在不同时期可以有多条记录（历史搬离+新入住），数据库不加唯一约束，由应用层逻辑控制

***

### 4.3 `user` — 用户登录表（纯认证）

| 字段名           | 类型                                     | 约束                         | 说明                                              |
| ------------- | -------------------------------------- | -------------------------- | ----------------------------------------------- |
| `user_id`     | INT                                    | **PK**, AUTO\_INCREMENT    | 用户ID                                            |
| `username`    | VARCHAR(50)                            | **NOT NULL**, **UNIQUE**   | 登录用户名                                           |
| `password`    | VARCHAR(200)                           | **NOT NULL**               | BCrypt加密密码                                      |
| `role`        | ENUM('admin','maintenance','resident') | **NOT NULL**               | 角色                                              |
| `ref_id`      | VARCHAR(50)                            | <br />                     | 关联ID：resident=住户ID，maintenance=维修员工号，admin=NULL |
| `status`      | TINYINT                                | DEFAULT 1, CHECK IN (0,1)  | 1=启用, 0=禁用                                      |
| `create_time` | DATETIME                               | DEFAULT CURRENT\_TIMESTAMP | 创建时间                                            |

**约束说明：**

- `username` 为 UNIQUE（含已禁用的账号），用户名全局唯一
- `ref_id` 根据 `role` 决定指向哪个表，不在数据库层面设外键
- 登录密码使用 **BCrypt** 加密存储，初始密码为 `123456`

**用户名命名规则（自动生成）：**

| 角色  | 格式              | 示例                       | 说明        |
| --- | --------------- | ------------------------ | --------- |
| 业主  | `yz-楼号层号户号-时间戳` | `yz-281302-202607041759` | 创建住户时自动生成 |
| 维修员 | `wx-时间戳`        | `wx-202607041759`        | 创建员工时自动生成 |
| 管理员 | 手动指定            | `admin`                  | 新增时管理员自己填 |

### 4.3+ `maintenance_staff` — 维修员信息表

| 字段名           | 类型          | 约束                         | 说明                   |
| ------------- | ----------- | -------------------------- | -------------------- |
| `worker_no`   | VARCHAR(20) | **PK**                     | 工号（如 WX202607041759） |
| `real_name`   | VARCHAR(50) | **NOT NULL**               | 真实姓名                 |
| `phone`       | VARCHAR(20) | <br />                     | 联系电话                 |
| `status`      | TINYINT     | DEFAULT 1                  | 1=在职, 0=离职           |
| `create_time` | DATETIME    | DEFAULT CURRENT\_TIMESTAMP | 入职时间                 |

**用户信息展示逻辑：** resident 的真实姓名和电话从 `household` 表取；maintenance 从 `maintenance_staff` 表取；admin 直接显示用户名。

***

### 4.4 `property_fee` — 物业费缴费记录表（按月）

| 字段名            | 类型            | 约束                                              | 说明            |
| -------------- | ------------- | ----------------------------------------------- | ------------- |
| `fee_id`       | INT           | **PK**, AUTO\_INCREMENT                         | 记录ID          |
| `household_id` | INT           | **FK** → household(household\_id), **NOT NULL** | 住户ID          |
| `year`         | INT           | **NOT NULL**                                    | 年份            |
| `month`        | INT           | **NOT NULL**, CHECK 1\~12                       | 月份            |
| `amount`       | DECIMAL(10,2) | **NOT NULL**, CHECK > 0                         | 应缴金额（元）       |
| `is_paid`      | TINYINT       | DEFAULT 0, CHECK IN (-1,0,1)                    | -1=逾期, 0=待缴, 1=已缴 |
| `pay_date`     | DATE          | <br />                                          | 缴费日期          |
| `handler`      | VARCHAR(50)   | <br />                                          | 经手人（管理员收费时填写） |
| `bill_no`      | VARCHAR(50)   | <br />                                          | 缴费单号          |

**约束说明：**

- **唯一约束：** `UNIQUE(household_id, year, month)` — 每户每月只有一条物业费记录
- **CHECK约束：** `amount > 0`, `month BETWEEN 1 AND 12`, `is_paid IN (-1,0,1)`（-1=逾期, 0=待缴, 1=已缴）
- 月应缴金额 = `household.area × household.property_fee_rate`

***

### 4.5 `parking_space` — 停车位信息表（space\_no 直接做主键）

| 字段名             | 类型            | 约束                                | 说明                         |
| --------------- | ------------- | --------------------------------- | -------------------------- |
| `space_no`      | VARCHAR(20)   | **PK**                            | 停车位编号（如 "A001"）            |
| `household_id`  | INT           | **FK** → household(household\_id) | 租用住户（NULL=空闲）              |
| `plate_no`      | VARCHAR(20)   | <br />                            | 车牌号                        |
| `monthly_fee`   | DECIMAL(10,2) | **NOT NULL**, CHECK > 0           | 月停车费（元）                    |
| `status`        | TINYINT       | DEFAULT 1, CHECK IN (0,1)         | 1=已租, 0=空闲                 |
| `create_time`   | DATETIME      | DEFAULT CURRENT\_TIMESTAMP        | 车位记录创建时间                   |
| `assigned_date` | DATE          | DEFAULT NULL                      | 当前租户的分配日期（用于判断计费起始月），释放时清空 |

***

### 4.6 `parking_fee` — 停车费缴费记录表（按月）

| 字段名            | 类型            | 约束                                               | 说明                |
| -------------- | ------------- | ------------------------------------------------ | ----------------- |
| `fee_id`       | INT           | **PK**, AUTO\_INCREMENT                          | 记录ID              |
| `space_no`     | VARCHAR(20)   | **FK** → parking\_space(space\_no), **NOT NULL** | 停车位编号             |
| `household_id` | INT           | **FK** → household(household\_id), **NOT NULL**  | **冻结**：生成费用时的住户ID |
| `year`         | INT           | **NOT NULL**                                     | 年份                |
| `month`        | INT           | **NOT NULL**, CHECK 1\~12                        | 月份                |
| `amount`       | DECIMAL(10,2) | **NOT NULL**, CHECK > 0                          | 应缴金额              |
| `is_paid`      | TINYINT       | DEFAULT 0, CHECK IN (-1,0,1)                     | -1=逾期, 0=待缴, 1=已缴 |
| `pay_date`     | DATE          | <br />                                           | 缴费日期              |
| `handler`      | VARCHAR(50)   | <br />                                           | 经手人               |
| `bill_no`      | VARCHAR(50)   | <br />                                           | 缴费单号              |

**约束说明：**

- **唯一约束：** `UNIQUE(space_no, year, month)` — 每个车位每月只有一条记录
- **CHECK约束：** `amount > 0`, `month BETWEEN 1 AND 12`, `is_paid IN (-1,0,1)`（-1=逾期, 0=待缴, 1=已缴）
- `household_id` 在生成费用时从 `parking_space.household_id` 复制，**冻结历史关联**。即使车位换租，历史费用仍指向原住户

**停车位换租策略：** 直接修改 `parking_space.household_id` 和 `plate_no`，不用新增记录

- 月应缴金额 = `parking_space.monthly_fee`

## 五、缴费流程设计

### 5.1 账单状态定义

缴费管理页面直接读取数据库账单表展示账单状态，以库内存储的状态为准：

| 状态值 | 状态名称 | 说明                              |
| :---: | :----- | ------------------------------- |
| `-1`  | 逾期    | 已过缴费截止日仍未缴纳，缴费前永久保持逾期状态       |
| `0`   | 待缴    | 未到截止日             |
| `1`   | 已缴    | 已完成缴费，状态不再变更                    |

### 5.2 账单生成规则

系统分**物业费**、**停车费**两类账单，生成规则如下：

| 费用类型 | 判定依据 | 生成规则                                  |
| ---- | ----- | ------------------------------------- |
| 物业费  | 房屋入住日期 | 只要当月完成入住，无论当月几号入住，都必须生成当月账单        |
| 停车费  | 车位租赁日期 | 只要当月完成车位租赁，无论当月几号租车位，都必须生成当月账单     |

#### 5.2.1 入住/租车位时自动生成当月账单

**物业费场景：**
```
管理员办理入住 → 填写 check_in_date（如 2026-07-15）
  → 判断入住日期与当月截止日关系
  → 若入住日期 ≤ 截止日 → 状态 = 待缴(0)
  → 若入住日期 > 截止日 → 状态 = 逾期(-1)
  → 直接插入当月物业费账单到 property_fee 表
```

**停车费场景：**
```
管理员分配车位 → 填写 assigned_date（如 2026-07-15）
  → 判断租赁日期与当月截止日关系
  → 若租赁日期 ≤ 截止日 → 状态 = 待缴(0)
  → 若租赁日期 > 截止日 → 状态 = 逾期(-1)
  → 直接插入当月停车费账单到 parking_fee 表
```

#### 5.2.2 每月1日定时任务补全账单

```
定时任务（每月1日 00:00:00）
  → 查询所有在住住户（household.status=1）
  → 检查本月是否已有物业费账单记录
  → 若无记录，则新增一条状态为【待缴(0)】的当月账单
  → 同理处理所有已租车位（parking_space.status=1）的停车费账单
```

### 5.3 逾期状态自动更新

**定时任务（每天 00:00:00）：**

```
检查当天是否为截止日后的第一天
  → 若当天 = 截止日 + 1（如截止日10号，当天11号）
    → 将所有状态为【待缴(0)】的账单更新为【逾期(-1)】
  → 账单一旦变为逾期，在用户完成缴费前永久保持逾期状态
```

### 5.4 缴费操作流程

#### 5.4.1 缴费周期支持

| 缴费周期 | 月份数 | 说明                  |
| ---- | --- | ------------------- |
| 1个月  | 1   | 当月或指定单月缴费           |
| 半年   | 6   | 连续6个月，从起始月开始        |
| 1年    | 12  | 连续12个月，从起始月开始       |

#### 5.4.2 缴费流程

```
业主选择缴费周期（1个月/半年/1年）
  → 查询所有逾期/待缴费用，按时间排序，先把时间最早的未缴费的交了，如果所有未缴费的月份都缴费了，还有没缴满选择的周期，就接着给未来月份缴费。
  → 生成同一个支付订单号（bill_no）
  → 对应多条分月账单记录（1条/6条/12条）
  → 一次性支付成功
```

**示例：半年缴费**
```
有4月、5月、6月逾期的费用，就应该缴费4~9月
结果：生成订单号 WY20260703150230001
对应：2026-04 ~ 2026-9 共 6 条账单记录
操作：6条记录共用同一个 bill_no，一次性全部标记已缴
```

### 5.5 缴费截止日管理

#### 5.5.1 修改规则

- 缴费截止日仅允许在当前月份内修改
- 不可修改至下一个月
- 范围限制：1~28日

#### 5.5.2 修改后的状态同步

修改当月截止日后，统一处理本月全部账单：

```
已缴(1)账单 → 不作任何变更

待缴(0)/逾期(-1)账单 → 基于新截止日、系统当前日期重新判定
  → 当前日期 ≤ 新截止日 → 更新为 待缴(0)
  → 当前日期 > 新截止日 → 更新为 逾期(-1)
```

### 5.6 未来月份账单处理

对于未来月份的缴费情况：

- 查询数据库是否有未来月份的缴费记录
- 有记录则直接显示（支持预缴场景）

### 5.7 缴费单号规则

| 费用类型 | 单号前缀 | 示例                    |
| ---- | ---- | --------------------- |
| 物业费  | WY   | `WY20260703150230001` |
| 停车费  | TC   | `TC20260703150230001` |

格式：`前缀 + yyyyMMddHHmmss + 3位随机数`

### 5.8 完整流程示例

#### 示例一：老住户张三（1月办理住），月物业费 350 元

>办理入住时，会生成当月物业缴费，并由入住日与截止日判断status是0/1/-1
若没有预缴后面月份，每月1日定时任务为张三生成当月物业费账单，初始状态为待缴(0)。

**场景A：7月5日缴纳当月费用（截止日10号）**

```
当前状态: is_paid=0（待缴）
金额: 350 元
操作: UPDATE property_fee SET is_paid=1, pay_date='2026-07-05', bill_no='WY20260705001'
       WHERE household_id=1 AND year=2026 AND month=7
结果: is_paid=1（已缴）
```

**场景B：7月12日缴纳当月费用（已逾期）**

```
定时任务（7月11日）已将 is_paid=0 更新为 is_paid=-1（逾期）
金额: 350 元
操作: UPDATE property_fee SET is_paid=1, pay_date='2026-07-12', bill_no='WY20260712001'
       WHERE household_id=1 AND year=2026 AND month=7
结果: is_paid=1（已缴），逾期状态在缴费后恢复为已缴
```

#### 示例二：新住户李四，2026年7月15日入住，月物业费 300 元，截止日10号

> 入住日期（15号）> 截止日（10号），因此当月账单状态为逾期。

```
管理员办理入住 → check_in_date=2026-07-15
判断: 15 > 10 → 状态 = 逾期(-1)
插入记录: property_fee(household_id=2, year=2026, month=7, amount=300, is_paid=-1)
```

#### 示例三：截止日修改场景

> 当前日期7月12日，原截止日10号，修改为15号。

```
原状态: 7月账单 is_paid=-1（逾期，已过10号）
修改截止日: 10号 → 15号
重新判定: 12 ≤ 15 → 更新为 待缴(0)
结果: 7月账单 is_paid=0（待缴）
```

## 七、存储过程设计

### 7.1 查询住户物业费及停车费

```sql
-- 输入：楼号、房号（层号+户号） → 输出该户物业费+停车费基本信息
DELIMITER //
CREATE PROCEDURE sp_query_household_fees(
  IN p_building_no VARCHAR(10),
  IN p_floor_no INT,
  IN p_unit_no INT
)
BEGIN
  -- 物业费缴费情况（状态：-1=逾期, 0=待缴, 1=已缴）
  SELECT 
    b.building_no AS 楼号,
    CONCAT(h.floor_no, LPAD(h.unit_no, 2, '0')) AS 房号,
    h.owner_name AS 户主,
    pf.year AS 年份,
    pf.month AS 月份,
    pf.amount AS 应缴物业费,
    CASE 
      WHEN pf.is_paid = 1 THEN '已缴'
      WHEN pf.is_paid = -1 THEN '逾期'
      WHEN pf.is_paid = 0 THEN '待缴'
    END AS 缴费状态,
    pf.pay_date AS 缴费日期,
    pf.bill_no AS 缴费单号
  FROM household h
  JOIN building b ON h.building_no = b.building_no
  LEFT JOIN property_fee pf ON h.household_id = pf.household_id
  WHERE b.building_no = p_building_no
    AND h.floor_no = p_floor_no
    AND h.unit_no = p_unit_no
    AND h.status = 1
  ORDER BY pf.year DESC, pf.month DESC;
  
  -- 停车费缴费情况（状态：-1=逾期, 0=待缴, 1=已缴）
  SELECT 
    ps.space_no AS 停车位编号,
    ps.plate_no AS 车牌号,
    pkf.year AS 年份,
    pkf.month AS 月份,
    pkf.amount AS 应缴停车费,
    CASE 
      WHEN pkf.is_paid = 1 THEN '已缴'
      WHEN pkf.is_paid = -1 THEN '逾期'
      WHEN pkf.is_paid = 0 THEN '待缴'
    END AS 缴费状态,
    pkf.pay_date AS 缴费日期,
    pkf.bill_no AS 缴费单号
  FROM household h
  JOIN building b ON h.building_no = b.building_no
  JOIN parking_space ps ON h.household_id = ps.household_id
  LEFT JOIN parking_fee pkf ON ps.space_no = pkf.space_no
  WHERE b.building_no = p_building_no
    AND h.floor_no = p_floor_no
    AND h.unit_no = p_unit_no
    AND h.status = 1
  ORDER BY pkf.year DESC, pkf.month DESC;
END//
DELIMITER ;
```

### 7.2 按月统计物业费汇总

```sql
-- 输入年、月 → 应缴总额、已缴总额、待缴总额、逾期总额
DELIMITER //
CREATE PROCEDURE sp_calc_property_fee_summary(
  IN p_year INT,
  IN p_month INT
)
BEGIN
  SELECT
    p_year AS 年份,
    p_month AS 月份,
    COUNT(*) AS 总户数,
    SUM(pf.amount) AS 应缴物业费总额,
    SUM(CASE WHEN pf.is_paid = 1 THEN pf.amount ELSE 0 END) AS 已缴物业费总额,
    SUM(CASE WHEN pf.is_paid = 0 THEN pf.amount ELSE 0 END) AS 待缴物业费总额,
    SUM(CASE WHEN pf.is_paid = -1 THEN pf.amount ELSE 0 END) AS 逾期物业费总额,
    COUNT(CASE WHEN pf.is_paid = 0 THEN 1 END) AS 待缴户数,
    COUNT(CASE WHEN pf.is_paid = -1 THEN 1 END) AS 逾期户数
  FROM property_fee pf
  JOIN household h ON pf.household_id = h.household_id
  WHERE pf.year = p_year AND pf.month = p_month AND h.status = 1;
END//
DELIMITER ;
```

### 7.4 每月1日生成当月账单（定时任务调用）

```sql
-- 为所有在住住户生成当月物业费账单，为所有已租车位生成当月停车费账单
-- 关键逻辑：通过 NOT EXISTS 检查避免重复生成
-- 若住户/车位已存在当月账单记录（包括预缴生成的记录），则跳过不重复生成
DELIMITER //
CREATE PROCEDURE sp_generate_monthly_bills()
BEGIN
  DECLARE current_year INT;
  DECLARE current_month INT;
  
  SELECT YEAR(CURDATE()) INTO current_year;
  SELECT MONTH(CURDATE()) INTO current_month;
  
  -- 生成物业费账单：为所有在住住户（status=1）生成当月记录
  -- NOT EXISTS 确保：若住户已预缴本月费用（is_paid=1）或已有待缴记录（is_paid=0/-1），则不再重复生成
  INSERT INTO property_fee (household_id, year, month, amount, is_paid)
  SELECT h.household_id, current_year, current_month,
         h.area * h.property_fee_rate, 0
  FROM household h
  WHERE h.status = 1
    AND NOT EXISTS (
      SELECT 1 FROM property_fee pf
      WHERE pf.household_id = h.household_id
        AND pf.year = current_year
        AND pf.month = current_month
    );
  
  -- 生成停车费账单：为所有已租车位（status=1）生成当月记录
  -- NOT EXISTS 确保：若车位已预缴本月费用或已有待缴记录，则不再重复生成
  INSERT INTO parking_fee (space_no, household_id, year, month, amount, is_paid)
  SELECT ps.space_no, ps.household_id, current_year, current_month,
         ps.monthly_fee, 0
  FROM parking_space ps
  WHERE ps.status = 1
    AND ps.household_id IS NOT NULL
    AND NOT EXISTS (
      SELECT 1 FROM parking_fee pf
      WHERE pf.space_no = ps.space_no
        AND pf.year = current_year
        AND pf.month = current_month
    );
END//
DELIMITER ;
```

### 7.5 每天检查逾期并更新状态（定时任务调用）

```sql
-- 检查当天是否为截止日后的第一天，若是则将所有待缴账单更新为逾期
-- 输入参数：p_deadline_day - 每月缴费截止日（如10表示每月10号）
DELIMITER //
CREATE PROCEDURE sp_mark_overdue(IN p_deadline_day INT)
BEGIN
  DECLARE current_day INT;
  
  SELECT DAY(CURDATE()) INTO current_day;
  
  -- 只有当天是截止日后的第一天时才执行逾期标记
  IF current_day = p_deadline_day + 1 THEN
    -- 更新物业费逾期状态
    UPDATE property_fee
    SET is_paid = -1
    WHERE is_paid = 0
      AND year = YEAR(CURDATE())
      AND month = MONTH(CURDATE());
    
    -- 更新停车费逾期状态
    UPDATE parking_fee
    SET is_paid = -1
    WHERE is_paid = 0
      AND year = YEAR(CURDATE())
      AND month = MONTH(CURDATE());
  END IF;
END//
DELIMITER ;
```

### 7.6 修改截止日后刷新当月账单状态

```sql
-- 修改缴费截止日后，重新判定当月所有待缴/逾期账单的状态
-- 输入参数：p_deadline_day - 新的每月缴费截止日
DELIMITER //
CREATE PROCEDURE sp_refresh_after_deadline_change(IN p_deadline_day INT)
BEGIN
  DECLARE current_day INT;
  DECLARE current_year INT;
  DECLARE current_month INT;
  
  SELECT DAY(CURDATE()) INTO current_day;
  SELECT YEAR(CURDATE()) INTO current_year;
  SELECT MONTH(CURDATE()) INTO current_month;
  
  -- 重新判定物业费账单状态
  UPDATE property_fee
  SET is_paid = CASE
    WHEN is_paid = 1 THEN 1  -- 已缴账单不变
    WHEN current_day <= p_deadline_day THEN 0  -- 当前日期未过新截止日，设为待缴
    ELSE -1  -- 当前日期已过新截止日，设为逾期
  END
  WHERE year = current_year
    AND month = current_month
    AND is_paid IN (0, -1);
  
  -- 重新判定停车费账单状态
  UPDATE parking_fee
  SET is_paid = CASE
    WHEN is_paid = 1 THEN 1  -- 已缴账单不变
    WHEN current_day <= p_deadline_day THEN 0  -- 当前日期未过新截止日，设为待缴
    ELSE -1  -- 当前日期已过新截止日，设为逾期
  END
  WHERE year = current_year
    AND month = current_month
    AND is_paid IN (0, -1);
END//
DELIMITER ;
```

## 九、催缴提醒逻辑

催缴功能在应用层实现，不等同于单独一张表，SQL 查询即可。账单状态以数据库存储为准（`is_paid` 字段：-1=逾期, 0=待缴, 1=已缴）。

**催缴规则（截止日可配置，默认为每月 10 号）：**

- **待缴提醒**：`is_paid = 0`（待缴） → 未到截止日
- **已逾期**：`is_paid = -1`（逾期） → 已过截止日仍未缴纳
- 只查当前月及以前未缴的记录，不查未来月份

**催缴判定示例（假设截止日为每月 10 号）：**

| 当前日期  | 缴费月份    | is_paid | 判定              |
| ----- | ------- | :-----: | --------------- |
| 7月5日  | 2026-07 |    0    | **待缴** — 还在截止日前 |
| 7月12日 | 2026-07 |   -1    | **已逾期** — 定时任务已更新状态 |
| 7月12日 | 2026-06 |   -1    | **已逾期** — 往月未缴  |
| 7月12日 | 2026-08 |   —     | 不显示 — 未来月份不催缴   |

**状态流转说明：**

```
待缴(0) → 逾期(-1)       // 定时任务（截止日后第一天）自动更新
待缴(0) → 已缴(1)        // 用户完成缴费
逾期(-1) → 已缴(1)       // 用户完成缴费（逾期状态在缴费前永久保持）
已缴(1) → 不再变更        // 缴费完成后状态固定
```
