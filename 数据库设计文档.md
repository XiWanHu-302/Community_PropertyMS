## 二、技术架构（中等档）

```
┌──────────────────────────────────────────────────┐

│              Nginx (反向代理)                      │
```

| `household` 1:N `parking_space` | 一户可租多个停车位 |

## 四、表结构详细设计

### 4.1 `building` — 楼栋信息表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `building_no` | VARCHAR(10) | **PK** | 楼号（如 "28"），唯一标识一栋楼 |
| `floor_count` | INT | **NOT NULL**, CHECK ≥ 1 | 总层数 |
| `units_per_floor` | INT | **NOT NULL**, CHECK ≥ 1 | 每层户数 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**约束说明：**
- `building_no` 作为主键，直接使用楼号，不另设自增ID
- `floor_count` 和 `units_per_floor` 必须 ≥ 1

**用途：** 房号合法性校验。如楼栋 28 号有 28 层、每层 4 户，则层号范围 1~28，户号范围 1~4。房号 "1302" 对应 13 层 02 户号，需满足 13 ≤ 28 且 2 ≤ 4。

---

### 4.2 `household` — 住户信息表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `household_id` | INT | **PK**, AUTO_INCREMENT | 住户ID |
| `building_no` | VARCHAR(10) | **FK** → building(building_no), **NOT NULL** | 所属楼栋（楼号） |
| `floor_no` | INT | **NOT NULL** | 所在层号 |
| `unit_no` | INT | **NOT NULL** | 户号（该层第几户） |
| `area` | DECIMAL(10,2) | **NOT NULL**, CHECK > 0 | 房屋面积（平方米） |
| `property_fee_rate` | DECIMAL(10,2) | **NOT NULL**, CHECK > 0 | 物业费单价（元/㎡/月） |
| `owner_name` | VARCHAR(50) | **NOT NULL** | 户主姓名 |
| `phone` | VARCHAR(20) | **NOT NULL** | 联系电话 |
| `work_unit` | VARCHAR(100) | | 工作单位 |
| `family_size` | INT | DEFAULT 1 | 家庭人数 |
| `repair_fund_balance` | DECIMAL(12,2) | DEFAULT 0 | 维修基金余额（元） |
| `status` | TINYINT | DEFAULT 1, CHECK IN (0,1) | 1=在住, 0=已搬离 |
| `check_in_date` | DATE | | 入住日期 |
| `check_out_date` | DATE | | 搬离日期 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**约束说明：**
- **CHECK约束：** `area > 0`, `property_fee_rate > 0`, `status IN (0,1)`
- **外键约束：** `building_no` 引用 `building(building_no)`，`ON DELETE RESTRICT` 禁止删除有住户的楼栋
- **房号显示：** 前端拼接显示为 `CONCAT(floor_no, LPAD(unit_no, 2, '0'))`，如 13楼2户号 → "1302"
- **房号合法性校验（应用层）：** `floor_no ≤ building.floor_count AND unit_no ≤ building.units_per_floor`
- **房间唯一性（应用层）：** 同一房间只允许一户**在住**（`status=1`），已搬离的不限制。因此同一房间在不同时期可以有多条记录（历史搬离+新入住），数据库不加唯一约束，由应用层逻辑控制

---

### 4.3 `user` — 用户登录表（纯认证）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `user_id` | INT | **PK**, AUTO_INCREMENT | 用户ID |
| `username` | VARCHAR(50) | **NOT NULL**, **UNIQUE** | 登录用户名 |
| `password` | VARCHAR(200) | **NOT NULL** | BCrypt加密密码 |
| `role` | ENUM('admin','maintenance','resident') | **NOT NULL** | 角色 |
| `ref_id` | VARCHAR(50) | | 关联ID：resident=住户ID，maintenance=维修员工号，admin=NULL |
| `status` | TINYINT | DEFAULT 1, CHECK IN (0,1) | 1=启用, 0=禁用 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**约束说明：**
- `username` 为 UNIQUE（含已禁用的账号），用户名全局唯一
- `ref_id` 根据 `role` 决定指向哪个表，不在数据库层面设外键
- 登录密码使用 **BCrypt** 加密存储，初始密码为 `123456`

**用户名命名规则（自动生成）：**

| 角色 | 格式 | 示例 | 说明 |
|------|------|------|------|
| 业主 | `yz-楼号层号户号-时间戳` | `yz-281302-202607041759` | 创建住户时自动生成 |
| 维修员 | `wx-时间戳` | `wx-202607041759` | 创建员工时自动生成 |
| 管理员 | 手动指定 | `admin` | 新增时管理员自己填 |

### 4.3+ `maintenance_staff` — 维修员信息表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `worker_no` | VARCHAR(20) | **PK** | 工号（如 WX202607041759） |
| `real_name` | VARCHAR(50) | **NOT NULL** | 真实姓名 |
| `phone` | VARCHAR(20) | | 联系电话 |
| `status` | TINYINT | DEFAULT 1 | 1=在职, 0=离职 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 入职时间 |

**用户信息展示逻辑：** resident 的真实姓名和电话从 `household` 表取；maintenance 从 `maintenance_staff` 表取；admin 直接显示用户名。

---

### 4.4 `property_fee` — 物业费缴费记录表（按月）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `fee_id` | INT | **PK**, AUTO_INCREMENT | 记录ID |
| `household_id` | INT | **FK** → household(household_id), **NOT NULL** | 住户ID |
| `year` | INT | **NOT NULL** | 年份 |
| `month` | INT | **NOT NULL**, CHECK 1~12 | 月份 |
| `amount` | DECIMAL(10,2) | **NOT NULL**, CHECK > 0 | 应缴金额（元） |
| `is_paid` | TINYINT | DEFAULT 0, CHECK IN (0,1) | 0=未缴, 1=已缴 |
| `pay_date` | DATE | | 缴费日期 |
| `handler` | VARCHAR(50) | | 经手人（管理员收费时填写） |
| `bill_no` | VARCHAR(50) | | 缴费单号 |

**约束说明：**
- **唯一约束：** `UNIQUE(household_id, year, month)` — 每户每月只有一条物业费记录
- **CHECK约束：** `amount > 0`, `month BETWEEN 1 AND 12`
- 月应缴金额 = `household.area × household.property_fee_rate`

---

### 4.5 `parking_space` — 停车位信息表（space_no 直接做主键）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `space_no` | VARCHAR(20) | **PK** | 停车位编号（如 "A001"） |
| `household_id` | INT | **FK** → household(household_id) | 租用住户（NULL=空闲） |
| `plate_no` | VARCHAR(20) | | 车牌号 |
| `monthly_fee` | DECIMAL(10,2) | **NOT NULL**, CHECK > 0 | 月停车费（元） |
| `status` | TINYINT | DEFAULT 1, CHECK IN (0,1) | 1=已租, 0=空闲 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 车位记录创建时间 |
| `assigned_date` | DATE | DEFAULT NULL | 当前租户的分配日期（用于判断计费起始月），释放时清空 |

---

### 4.6 `parking_fee` — 停车费缴费记录表（按月）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `fee_id` | INT | **PK**, AUTO_INCREMENT | 记录ID |
| `space_no` | VARCHAR(20) | **FK** → parking_space(space_no), **NOT NULL** | 停车位编号 |
| `household_id` | INT | **FK** → household(household_id), **NOT NULL** | **冻结**：生成费用时的住户ID |
| `year` | INT | **NOT NULL** | 年份 |
| `month` | INT | **NOT NULL**, CHECK 1~12 | 月份 |
| `amount` | DECIMAL(10,2) | **NOT NULL**, CHECK > 0 | 应缴金额 |
| `is_paid` | TINYINT | DEFAULT 0, CHECK IN (0,1) | 0=未缴, 1=已缴 |
| `pay_date` | DATE | | 缴费日期 |
| `handler` | VARCHAR(50) | | 经手人 |
| `bill_no` | VARCHAR(50) | | 缴费单号 |

**约束说明：**
- **唯一约束：** `UNIQUE(space_no, year, month)` — 每个车位每月只有一条记录
- `household_id` 在生成费用时从 `parking_space.household_id` 复制，**冻结历史关联**。即使车位换租，历史费用仍指向原住户

**停车位换租策略：** 直接修改 `parking_space.household_id` 和 `plate_no`，不用新增记录
- 月应缴金额 = `parking_space.monthly_fee`
- 释放车位时，当前租用信息自动写入 `parking_space_history` 表

---

### 4.7 `parking_space_history` — 车位租用历史记录表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | INT | **PK**, AUTO_INCREMENT | 记录ID |
| `space_no` | VARCHAR(20) | **FK** → parking_space(space_no), ON DELETE CASCADE | 停车位编号 |
| `household_id` | INT | **NOT NULL** | 住户ID（冻结快照） |
| `plate_no` | VARCHAR(20) | | 当时的车牌号 |
| `owner_name` | VARCHAR(50) | | 冻结户主姓名（释放时快照） |
| `room` | VARCHAR(20) | | 冻结房号（释放时快照） |
| `monthly_fee` | DECIMAL(10,2) | **NOT NULL** | 当时的月费 |
| `assigned_date` | DATE | **NOT NULL** | 租用开始日期 |
| `released_date` | DATE | **NOT NULL** | 释放日期 |

**说明：**
- 每次释放车位时自动写入一条记录，用于追溯车位历史租用情况
- `owner_name` 和 `room` 在释放时通过 OpenFeign 快照获取，不受后续住户信息变更影响
- 外键 `ON DELETE CASCADE`：删除车位时自动删除其历史记录
- 只检查这个月以前的未缴费，这个月的未缴费不检查。（要不要改成这个月也要检查）

### 4.8 `repair` — 维修信息表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `repair_id` | INT | **PK**, AUTO_INCREMENT | 维修ID |
| `household_id` | INT | **FK** → household(household_id), **NOT NULL** | 报修住户 |
| `content` | VARCHAR(500) | **NOT NULL** | 维修内容描述 |
| `report_date` | DATE | **NOT NULL** | 报修日期 |
| `repair_date` | DATE | | 维修完成日期 |
| `amount` | DECIMAL(10,2) | CHECK ≥ 0 | 维修金额（元） |
| `is_from_fund` | TINYINT | DEFAULT 0, CHECK IN (0,1) | 是否从维修基金支出 |
| `repair_person` | VARCHAR(50) | | 维修人 |
| `status` | TINYINT | DEFAULT 0, CHECK IN (0,1,2) | 0=待维修, 1=已完成, 2=已取消 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | |

**约束说明：**
- **CHECK约束：** `repair_date >= report_date`（维修日期不能早于报修日期，允许当天完成，通过触发器或应用层校验）
- `amount ≥ 0`

## 五、缴费流程设计

### 5.1 整体流程

费用记录的生成时机有两种：

**① 新住户入住时（管理员操作）**

```
管理员登记入住 → 填写 check_in_date（如 2026-10-15）
  → 自动生成从入住月到当年12月的费用记录（即 10~12月，共3条）
  → 往月（1~9月）不生成，与新房主无关
```

**② 每年年初（管理员批量操作）**

```
管理员执行"生成年度费用" → 为所有在住住户（status=1）
  → 生成全年 1~12 月的费用记录
  → 已有记录的月份跳过（避免重复）
```

**③ 缴费环节（业主/管理员）**

```
业主看到自己名下的待缴清单 → 勾选任意连续月份
  → 1个月 / 半年6个月 / 一年12个月
  → 一次性支付 → 批量 UPDATE，共用同一个 bill_no
```

### 5.2 关键设计

| 问题 | 答案 |
|------|------|
| 谁来生成费用记录？ | **管理员**，两个时机：①新住户入住 ②每年年初批量 |
| 10月入住的，1~9月要生成吗？ | **不生成**，只从入住当月开始，到当年12月为止 |
| 怎么避免重复生成？ | 生成前检查：`property_fee` 已有 `(household_id, year, month)` 的记录跳过 |
| 缴费本质是什么？ | 不是 INSERT 新记录，而是 **UPDATE** 已有记录 → `SET is_paid=1, bill_no=xxx` |
| 批量怎么区分？ | 同一个 bill_no 管多个月，如 6 个月共用一个单号 |

### 5.3 缴费单号规则

| 费用类型 | 单号前缀 | 示例 |
|----------|----------|------|
| 物业费 | WY | `WY20260703150230001` |
| 停车费 | TC | `TC20260703150230001` |

格式：`前缀 + yyyyMMddHHmmss + 3位随机数`

### 5.4 完整流程示例

#### 示例一：老住户张三（1月就在住），月物业费 350 元

> 2026年初，管理员为张三生成全年 1~12 月共 12 条记录。

**场景A：1月份来交当月**

```
勾选: 2026-01（1条）
金额: 350 元
操作: UPDATE property_fee SET is_paid=1, pay_date='2026-01-05', bill_no='WY20260105001', handler='管理员李四'
       WHERE household_id=1 AND year=2026 AND month=1
```

**场景B：张三 1 月份来，一次交半年**

```
勾选: 2026-01 ~ 2026-06（6条）
金额: 350 × 6 = 2100 元
操作: UPDATE property_fee SET is_paid=1, pay_date='2026-01-05', bill_no='WY20260105001', handler='管理员李四'
       WHERE household_id=1 AND year=2026 AND month BETWEEN 1 AND 6
```

**场景C：张三 7 月份来，一次交下半年**

```
勾选: 2026-07 ~ 2026-12（6条）
金额: 350 × 6 = 2100 元
操作: UPDATE property_fee SET is_paid=1, pay_date='2026-07-03', bill_no='WY20260703001', handler='在线缴费'
       WHERE household_id=1 AND year=2026 AND month BETWEEN 7 AND 12
```

**场景D：张三一次交全年**

```
勾选: 2026-01 ~ 2026-12（12条）
金额: 350 × 12 = 4200 元
操作: 12 条记录共用同一个 bill_no，一次性全部标记已缴
```

#### 示例二：新住户李四，2026年10月入住，月物业费 300 元

> 管理员登记入住后，只生成 10~12 月共 3 条记录，往月不生成。

| household_id | year | month | amount | is_paid | 说明 |

## 七、存储过程设计

### 7.1 查询住户物业费及停车费

```sql
-- 输入：楼号、房号（层号+户号） → 输出该户物业费+停车费基本信息
DELIMITER //
CREATE PROCEDURE sp_query_household_fees(
  IN p_building_no VARCHAR(10),
  IN p_floor_no INT,
  IN p_unit_no INT
)
BEGIN
  -- 物业费缴费情况
  SELECT 
    b.building_no AS 楼号,
    CONCAT(h.floor_no, LPAD(h.unit_no, 2, '0')) AS 房号,
    h.owner_name AS 户主,
    pf.year AS 年份,
    pf.month AS 月份,
    pf.amount AS 应缴物业费,
    CASE WHEN pf.is_paid = 1 THEN '已缴' ELSE '未缴' END AS 缴费状态,
    pf.pay_date AS 缴费日期,
    pf.bill_no AS 缴费单号
  FROM household h
  JOIN building b ON h.building_no = b.building_no
  LEFT JOIN property_fee pf ON h.household_id = pf.household_id
  WHERE b.building_no = p_building_no
    AND h.floor_no = p_floor_no
    AND h.unit_no = p_unit_no
    AND h.status = 1
  ORDER BY pf.year DESC, pf.month DESC;
  
  -- 停车费缴费情况
  SELECT 
    ps.space_no AS 停车位编号,
    ps.plate_no AS 车牌号,
    pkf.year AS 年份,
    pkf.month AS 月份,
    pkf.amount AS 应缴停车费,
    CASE WHEN pkf.is_paid = 1 THEN '已缴' ELSE '未缴' END AS 缴费状态,
    pkf.pay_date AS 缴费日期,
    pkf.bill_no AS 缴费单号
  FROM household h
  JOIN building b ON h.building_no = b.building_no
  JOIN parking_space ps ON h.household_id = ps.household_id
  LEFT JOIN parking_fee pkf ON ps.space_id = pkf.space_id
  WHERE b.building_no = p_building_no
    AND h.floor_no = p_floor_no
    AND h.unit_no = p_unit_no
    AND h.status = 1
  ORDER BY pkf.year DESC, pkf.month DESC;
END//
DELIMITER ;
```

### 7.2 按月统计物业费汇总

```sql
-- 输入年、月 → 应缴总额、已缴总额、未缴总额
DELIMITER //
CREATE PROCEDURE sp_calc_property_fee_summary(
  IN p_year INT,
  IN p_month INT
)
BEGIN
  SELECT
    p_year AS 年份,
    p_month AS 月份,
    COUNT(*) AS 总户数,
    SUM(pf.amount) AS 应缴物业费总额,
    SUM(CASE WHEN pf.is_paid = 1 THEN pf.amount ELSE 0 END) AS 已缴物业费总额,
    SUM(CASE WHEN pf.is_paid = 0 THEN pf.amount ELSE 0 END) AS 未缴物业费总额,
    COUNT(CASE WHEN pf.is_paid = 0 THEN 1 END) AS 未缴户数
  FROM property_fee pf
  JOIN household h ON pf.household_id = h.household_id
  WHERE pf.year = p_year AND pf.month = p_month AND h.status = 1;
END//
DELIMITER ;
```

## 九、催缴提醒逻辑

催缴功能在应用层实现，不等同于单独一张表，SQL 查询即可。

**催缴规则（月初缴费，每月 10 号为截止日，可配置）：**
- **待缴提醒**：当月未缴 + 当前日期 ≤ 10 号 → 临近截止日，提醒缴纳
- **已逾期**：当月已过 10 号未缴，或往月未缴 → 已逾期
- 只查当前月及以前未缴的记录，不查未来月份

```sql
-- 物业费催缴（截止日：每月10号）
SET @deadline_day = 10;  -- 可配置的每月截止日

SELECT h.household_id, h.owner_name, h.phone,
       b.building_no, CONCAT(h.floor_no, LPAD(h.unit_no, 2, '0')) AS room_no,
       pf.year, pf.month, pf.amount, '物业费' AS fee_type,
       CASE 
         -- 当前日期已超过缴费月截止日 → 已逾期
         WHEN CURDATE() > CONCAT(pf.year, '-', LPAD(pf.month, 2, '0'), '-', @deadline_day)
           THEN '已逾期'
         -- 当前月份 + 未到截止日 → 待缴提醒
         ELSE '待缴'
       END AS remind_type
FROM property_fee pf
JOIN household h ON pf.household_id = h.household_id
JOIN building b ON h.building_no = b.building_no
WHERE pf.is_paid = 0
  AND h.status = 1
  -- 只查当前月及以前未缴的记录（不查未来月份）
  AND CONCAT(pf.year, '-', LPAD(pf.month, 2, '0'), '-01') <= CURDATE()
ORDER BY pf.year, pf.month;

-- 停车费催缴（逻辑相同）
SELECT h.household_id, h.owner_name, h.phone,
       b.building_no, CONCAT(h.floor_no, LPAD(h.unit_no, 2, '0')) AS room_no,
       ps.space_no, ps.plate_no,
       pkf.year, pkf.month, pkf.amount, '停车费' AS fee_type,
       CASE 
         WHEN CURDATE() > CONCAT(pkf.year, '-', LPAD(pkf.month, 2, '0'), '-', @deadline_day)
           THEN '已逾期'
         ELSE '待缴'
       END AS remind_type
FROM parking_fee pkf
JOIN parking_space ps ON pkf.space_id = ps.space_id
JOIN household h ON ps.household_id = h.household_id
JOIN building b ON h.building_no = b.building_no
WHERE pkf.is_paid = 0
  AND h.status = 1
  AND CONCAT(pkf.year, '-', LPAD(pkf.month, 2, '0'), '-01') <= CURDATE()
ORDER BY pkf.year, pkf.month;
```

**催缴判定示例（假设截止日为每月 10 号）：**

| 当前日期 | 缴费月份 | 是否已缴 | 判定 |
|----------|----------|----------|------|
| 7月5日 | 2026-07 | 否 | **待缴** — 还在截止日前 |
| 7月12日 | 2026-07 | 否 | **已逾期** — 已过10号 |
| 7月12日 | 2026-06 | 否 | **已逾期** — 往月未缴 |
| 7月12日 | 2026-08 | — | 不显示 — 未来月份不催缴 |

1. 系统分物业费、停车费两类账单：物业费依据房屋入住日期判定，停车费依据车位租赁日期判定；只要当月完成入住 / 租车位，无论当月几号入住，都必须生成当月账单
2. 本月记录，只要住户还在住，查数据库表本月账单记录，若无本月账单、且当前日期处于本月 1 日至本缴费截止日内，则新增一条状态为【待缴】的当月账单入库。
   3.逾期规则：账单超过当月缴费截止日仍未缴费，账单状态更新为【逾期】；账单一旦变为逾期，在用户完成缴费前，永久保持逾期状态
   4.业主办理入住、车位租赁时，实时根据入住 / 租车位日期、当月缴费截止日判断账单状态（待缴 / 逾期），直接插入当月账单记录。
5. 支持缴费周期：1 个月、半年、1 年；选择半年缴费时，生成同一个支付订单号，对应 6 条分月账单记录；
6. 缴费截止日仅允许在当前月份内修改，不可修改至下一个月；
7. 修改当月截止日后，统一处理本月全部账单：
   状态为【已缴】的账单，不作任何变更；
   所有【待缴 / 逾期】账单，基于新截止日、系统当前日期重新判定，同步更新账单状态。
8. 缴费管理页面，直接读取数据库账单表展示账单状态，以库内存储的状态为准。
   9.对于未来月份的缴费情况只显示已缴费的，并提示是提前缴费的记录，还未到缴费时间。
