## 作业五：微服务架构综合实践（Spring Cloud 扩展）

**课程名称：** JavaEE  
**适用对象：** 计算机科学与技术（含卓越班）课学生  
**技术栈：** Spring Boot 3.2.5 + Spring Cloud + Spring Cloud Gateway + Nacos + OpenFeign  
**前置要求：** 已完成作业三和作业四，具备用户管理和课程管理的基础代码

---

### 一、作业背景

目前我们的项目包含两个模块：`demo`（用户与课程管理服务，端口 8080）和 `gateway`（API 网关，端口 8081），两者都注册到 Nacos 注册中心。但所有业务逻辑集中在一个服务中，属于典型的单体架构。

本次作业要求同学们新建一个**选课服务（`enrollment-service`）**，实现跨服务的微服务调用，并完成网关路由配置、服务间通信、熔断降级等微服务核心能力，最终形成一个包含三个服务的完整微服务架构。

**目标架构：**

```
                    ┌──────────────┐
                    │   Gateway    │  端口 8081
                    │  (API 网关)   │
                    └──────┬───────┘
                           │ 路由转发
              ┌────────────┼────────────┐
              ▼            ▼            ▼
      ┌──────────┐ ┌──────────┐ ┌──────────────┐
      │   demo   │ │   demo   │ │  enrollment  │
      │ (用户服务) │ │ (课程服务) │ │  (选课服务)   │
      │  端口8080 │ │  端口8080 │ │  端口8082     │
      └──────────┘ └──────────┘ └──────────────┘
              │                        │
              └────── OpenFeign ───────┘
                      (服务间调用)
```

#### 2.2 数据库设计

创建选课记录表 `enrollment`：

```sql
CREATE TABLE IF NOT EXISTS enrollment (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    course_id BIGINT NOT NULL COMMENT '课程ID',
    score DECIMAL(5,2) COMMENT '成绩',
    status INT DEFAULT 0 COMMENT '选课状态：0-已选课，1-已结课，2-已退课',
    enroll_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '选课时间',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted INT DEFAULT 0 COMMENT '逻辑删除：0-未删除，1-已删除',
    UNIQUE KEY uk_user_course (user_id, course_id, deleted)
);
```

**注意：** `uk_user_course` 唯一索引确保同一用户不能重复选同一门课程。

#### 3.2 在 enrollment 服务中定义 Feign Client

```java
// UserFeignClient.java
@FeignClient(name = "demo", contextId = "userFeignClient", fallbackFactory = UserFeignFallbackFactory.class)
public interface UserFeignClient {

    @GetMapping("/internal/users")
    Result<List<UserInfo>> getUsersByIds(@RequestParam("ids") List<Long> ids);
}
```

```java
// CourseFeignClient.java
@FeignClient(name = "demo", contextId = "courseFeignClient", fallbackFactory = CourseFeignFallbackFactory.class)
public interface CourseFeignClient {

    @GetMapping("/internal/courses")
    Result<List<CourseInfo>> getCoursesByIds(@RequestParam("ids") List<Long> ids);
}
```

**注意：**
- 在 enrollment 服务中定义 `UserInfo` 和 `CourseInfo` 作为远程调用的数据接收类（只需包含 `id`、`username`/`courseName` 等必要字段）。
- 两个 Feign Client 都指向 `demo` 服务，必须使用 `contextId` 区分，否则 Spring 会报 Bean 名称冲突错误。

#### 3.3 Fallback 降级实现

为每个 Feign Client 编写 `FallbackFactory` 实现类，当远程服务不可用时返回降级数据：

```java
@Component
public class UserFeignFallbackFactory implements FallbackFactory<UserFeignClient> {
    @Override
    public UserFeignClient create(Throwable cause) {
        return ids -> {
            // 记录日志
            log.error("调用用户服务失败: {}", cause.getMessage());
            // 返回降级响应
            Result<List<UserInfo>> result = new Result<>();
            result.setCode(503);
            result.setMessage("用户服务暂时不可用，请稍后重试");
            return result;
        };
    }
}
```

### 四、任务三：网关路由配置（20 分）

#### 4.1 添加选课服务路由

在网关的 Nacos 配置（或本地 `application.yml`）中添加选课服务的路由规则：

```yaml
spring:
  cloud:
    gateway:
      routes:
        # 用户和课程服务（原有）
        - id: demo-service
          uri: lb://demo
          predicates:
            - Path=/api/users/**, /api/courses/**, /api/login, /api/refresh-token
          filters:
            - StripPrefix=1
        # 选课服务（新增）
        - id: enrollment-service
          uri: lb://enrollment
          predicates:
            - Path=/api/enrollments/**
          filters:
            - StripPrefix=1
```

#### 4.2 全局跨域配置

在网关中添加全局 CORS 配置，允许前端跨域访问：

```yaml
spring:
  cloud:
    gateway:
      globalcors:
        corsConfigurations:
          '[/**]':
            allowedOriginPatterns: "*"
            allowedMethods: "*"
            allowedHeaders: "*"
            allowCredentials: true
            maxAge: 3600
```

#### 4.3 请求限流（可选加分项）

使用 Gateway 内置的 `RequestRateLimiter` 过滤器，对 `/api/login` 路径做限流（每秒最多 5 个请求）：

```yaml
- id: login-rate-limit
  uri: lb://demo
  predicates:
    - Path=/api/login
  filters:
    - name: RequestRateLimiter
      args:
        redis-rate-limiter.replenishRate: 5
        redis-rate-limiter.burstCapacity: 10
        key-resolver: "#{@ipKeyResolver}"
```

### 八、参考提示

- 启动类 `EnrollmentApplication` 上需添加 `@EnableFeignClients` 注解以启用 Feign。
- OpenFeign 的 `@FeignClient(name = "demo")` 中 `name` 值必须与 Nacos 注册的服务名一致。
- 由于 demo 服务同时提供用户和课程两个领域的接口，Feign Client 中可以使用不同的 `path` 前缀区分（或在 demo 中做路径规划）。
- Sentinel 熔断需要启动 Sentinel Dashboard（`java -jar sentinel-dashboard.jar`），如环境不具备，可先用 `try-catch` 做简单降级处理。
- 网关路由中的 `StripPrefix=1` 表示去除路径中的第一段（如 `/api/enrollments/add` → `/enrollments/add`），请根据自己的路径设计调整。
- 测试时建议先绕过网关直接调用各服务接口确认功能正常，再通过网关进行端到端测试。
- 如果 Nacos 环境不可用，可使用 `spring.cloud.nacos.discovery.enabled=false` 配合 `spring.cloud.loadbalancer` 的静态配置做本地调试。

### 九、常见问题 FAQ

**Q1：Feign 调用时报 "No Feign Client for loadBalancing defined" 错误**  
A：确保引入了 `spring-cloud-starter-loadbalancer` 依赖，Spring Cloud 2023 版本已移除 Ribbon。

**Q2：Feign 调用返回的对象字段全部为 null**  
A：检查远程服务返回的 `Result<T>` 中的泛型类与本地定义的接收类字段名是否一致，建议两端使用相同字段命名。

**Q3：网关路由不生效，请求返回 404**  
A：检查 Nacos 中服务是否注册成功，路由配置中的 `uri: lb://服务名` 是否与 Nacos 中的服务名一致。

**Q4：两个 Feign Client 指向同一服务时报 "BeanDefinitionOverrideException"**  
A：当多个 `@FeignClient` 的 `name` 相同时，必须添加 `contextId` 属性区分，如 `@FeignClient(name = "demo", contextId = "userFeignClient")`。

**Q5：Sentinel Dashboard 无法连接**  
A：确保添加了 `-Dcsp.sentinel.dashboard.server=localhost:8080` JVM 参数或配置了 `spring.cloud.sentinel.transport.dashboard`，且 Dashboard 已启动。
